JDEE 2.4.0 Release Notes

This is a "catchup" release that contains many patches and other contributions that have been queuing up over the last few years. While there are many new JDK 1.5 features added, the next (2.4.1) release will contain more or hopefully the remainder of those language changes.
  • The old CVS Log entries have now moved to doc/history/cvs-revision-log.xml.
    Thanks to paullandes
  • The release version is now dynamically updated in the source at build time.
    Thanks to paullandes
  • Support for 1.5 generics and simplified for loops in jde-complete
    Thanks to Ke Lu, Przemystaw Wojnowski
  • More 1.5 and 1.6 support.
    Thanks to lentrig
  • jde-parse-get-buffer-class is now interactive and also returns class names sans packages.
    Thanks to Paul Landes
  • New addition of class driven bookmarks (jde-bookmark.el).
    Thanks to Paul Landes
  • More fontification (i.e. constructors).
    Thanks to Paul Landes
  • Beanshell can optionally display a separate buffer for errors based on customized variable `jde-bsh-separate-buffer'.
    Thanks to Paul Landes
  • New function `jde-parse-class-name' parses class name into package name parts so we don't have to pass around fully qualified type flags to functions that need this--i.e. source finding functions.
    Thanks to Paul Landes
  • New function `jde-read-class' provides better user input class finding.
    Thanks to Paul Landes
  • New function `jde-exception-goto' provides a way to go to a file and line number from an exception stack trace line..
    Thanks to Paul Landes
  • More robust jde-eval, which now doesn't get confused with extraneous show() output. This feature can be turned off..
    Thanks to Paul Landes
  • More parsing ability (i.e. reverse camel notation parsing, which replaces some key bindings, convert from reverse camel notation to other variable formats like SQL, Java properties, hungarian notation, C constants, etc. parse Beanshell/Java expressions).
    Thanks to Paul Landes
